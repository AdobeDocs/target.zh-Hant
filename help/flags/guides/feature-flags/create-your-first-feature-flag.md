# 建立您的第一個功能標幟 {#create-feature-flag}

## 先決條件 {#prerequisites}

在建立功能標幟之前，請完成下列工作：

* 您可以存取旗標主控台 — 請參閱[登入主控台](../console/log-in-to-the-console.md)
* 您的應用程式已上線 — 請參閱[將應用程式上線](../applications/onboard-your-application.md)
* 您有&#x200B;**開發人員**&#x200B;或&#x200B;**產品發行擁有者**&#x200B;角色

## 步驟1：建立功能標幟 {#create}

若要建立新功能標幟，請遵循主控台中的下列步驟：

1. 登入旗標主控台，並瀏覽至&#x200B;**功能與版本>功能旗標**。
2. 從&#x200B;**應用程式**&#x200B;下拉式清單中選取您的應用程式。
3. 選取&#x200B;**新功能**。
4. 提供標題、索引鍵、說明，以及選用的標籤。
5. 選擇性地新增對象條件（請參閱步驟2）。
6. 儲存您的功能標幟設定。

## 步驟2：新增對象條件 {#audience}

對象條件可控制哪些使用者會看到此功能。 您可以根據以下條件新增條件：

* 設定檔屬性（例如國家/地區、電子郵件網域、使用者ID）
* 內容變數
* 預先定義的對象區段

若要新增對象條件，請在建立或編輯功能標幟時前往&#x200B;**對象**&#x200B;標籤。

>[!NOTE]
>
>**開發人員**&#x200B;角色是沙箱式。 開發人員只能透過在&#x200B;**對象>設定檔>使用者ID**&#x200B;下新增自己的使用者ID，來公開功能。 若要向外部使用者公開功能，您需要&#x200B;**產品發行擁有者**&#x200B;角色。

## 步驟3：計算對象規模 {#audience-size}

新增對象條件後，選取底部列中的&#x200B;**計算**&#x200B;以取得符合功能資格的預估使用者人數。 這有助於在上線之前驗證您的目標定位。

## 步驟4：排程（選擇性） {#schedule}

您可以使用功能標幟設定中的&#x200B;**排程**&#x200B;選項，將功能標幟排程為在未來日期和時間啟用。

## 常見問題集：我無法將功能標幟新增為開發人員 {#faq}

**開發人員**&#x200B;角色是沙箱式。 開發人員可以將其使用者ID新增至對象，私下測試功能。 他們無法將功能公開給外部使用者。 使用&#x200B;**產品發行擁有者**&#x200B;角色將功能發行給外部使用者。 請聯絡您的團隊管理員以更新您的角色。
