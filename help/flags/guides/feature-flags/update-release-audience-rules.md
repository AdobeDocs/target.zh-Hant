# 更新發行對象規則 {#update-release-audience-rules}

## 存取對象設定 {#access}

若要開始設定接收您的版本的對象，請導覽至主控台中的對象設定：

1. 在Flags主控台中開啟您的發行版本。
2. 前往&#x200B;**對象**&#x200B;標籤。
3. 啟用&#x200B;**對象規則**&#x200B;旁的切換。
4. 選取要新增&#x200B;**包含規則** （誰應該收到版本）或&#x200B;**排除規則** （誰不該收到）。

## 支援的對象條件 {#criteria}

### ID type {#id-type}

根據使用者的帳戶型別（例如個人或企業帳戶）鎖定使用者。

### 國家 {#country}

根據註冊國家/地區來鎖定使用者。

### 使用者 ID {#user-id}

依唯一使用者識別碼(GUID)鎖定特定使用者。

### 電子郵件地址 {#email}

依使用者的確切電子郵件地址鎖定使用者。 若要一次新增多個電子郵件地址，請從運運算元下拉式清單中選取&#x200B;**in** （而非&#x200B;**is**），然後：

* 在文字方塊中輸入以逗號分隔的地址清單，或
* 每行上傳一個位址的`.txt`檔案，或
* 上傳包含地址清單的`.csv`檔案

如果您的清單非常大，請將其分割為較小的批次。

### 電子郵件網域 {#email-domain}

將電子郵件地址屬於特定網域（例如`yourcompany.com`）的所有使用者設為目標。

### 使用者端IP {#client-ip}

根據使用者端IP位址鎖定使用者。

### 百分比 {#percentage}

鎖定所有使用者的隨機百分比，包括未驗證的使用者。

### 百分比（僅限驗證） {#percentage-auth}

僅定位已驗證使用者的隨機百分比。 會排除未驗證的使用者。

## 結合多個規則 {#combining-rules}

您可以使用AND/OR邏輯來結合多個對象條件。

**範例:**

* 若要只鎖定特定網域的企業帳戶，請使用AND將&#x200B;**ID型別**&#x200B;規則與&#x200B;**電子郵件網域**&#x200B;規則結合。
* 若要鎖定特定國家/地區的10%使用者，請將&#x200B;**國家/地區**&#x200B;規則與&#x200B;**百分比**&#x200B;規則結合。

使用&#x200B;**+/-**&#x200B;圖示來新增或移除條件。 若要複製條件，請選取現有規則旁的復製圖示。 針對複雜的巢狀邏輯，請啟用&#x200B;**巢狀邏輯**，並在文字方塊中輸入有效的運算式。
