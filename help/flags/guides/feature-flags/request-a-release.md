# 要求發行 {#request-a-release}

## 先決條件 {#prerequisites}

* 您有&#x200B;**發行管理員**&#x200B;角色 — 請參閱<!-- broken link[User roles](../teams/user-roles.md) -->
* 您的應用程式已上線 — 請參閱[將應用程式上線](../applications/onboard-your-application.md)

>[!TIP]
>
>在要求發行之前，請檢閱<!--[Releases and cross-team feature groups](releases-and-cross-team-feature-groups.md)-->。 跨團隊功能群組能夠以較少的經常性費用滿足您的需求 — 這是自助式的，可支援更豐富的對象目標定位。

## 提交支援要求 {#submit}

發行版本建立並非自助式。 聯絡旗標支援以請求新版本。 提供下列資訊：

| 欄位 | 說明 |
|---|---|
| **名稱** | 此版本的唯一名稱。 請勿在發行版本名稱中包含空格。 |
| **團隊** | 將擁有此版本的團隊。 |
| **環境** | 應建立發行版本的環境（預備或生產）。 |
| **目標** | 此版本將推出哪些功能及推出原因的簡短說明。 |
| **應用程式** | 要包含在發行版本中的應用程式。 |
| **持續時間** | 此版本需要維持作用中狀態多久。 預計發行會在三個月內關閉或稽核。 如果您需要更長的時間，請提供業務理由。 |

>[!NOTE]
>
>任何指定時間的作用中發行版本數量有限。 計畫在三個月內稽核或關閉您的版本，以釋出容量給其他團隊。

## 建立版本後 {#after}

在收到已建立版本的確認後，請登入主控台並完成版本設定。 如需完整的步驟順序，請參閱[端對端發行工作流程](release-workflow-end-to-end.md)。
