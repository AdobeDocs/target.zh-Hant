# 標幟模式 {#modes}

標幟提供兩種不同的功能鎖定目標模式。 每個版本都有不同的發行控制用途，且是針對特定型別的應用程式或使用案例而設計。

>[!TIP]
>
>根據您控制的專案選擇模式：個別使用者工作階段或組織和環境範圍。

## 使用者層級鎖定目標 {#user-level}

### 概觀 {#user-level-overview}

使用者層級的鎖定目標可讓您在個別使用者層級轉出功能。 它支援精確鎖定特定使用者、內部測試者、Canary使用者，以及以百分比為基礎的逐步推出。

此模式最適合需要根據登入者來改變體驗的應用程式。

### 使用時機 {#user-level-when}

當您想要執行下列動作時使用使用者層級鎖定目標：

* 為特定使用者啟用功能
* 執行A/B實驗
* 在更廣泛的推出之前，先對小型群組執行Canary測試
* 依使用者百分比逐步推出功能
* 根據個別使用者屬性（例如電子郵件網域或設定檔資料）微調體驗

### 限制 {#user-level-limitations}

使用者層級鎖定目標並非針對下列情況而設計：

* 無法控制組織、環境或區域層級的功能
* 不適用於租使用者範圍功能啟用或權益閘道

## 組織和環境層級鎖定目標 {#org-level}

### 概觀 {#org-level-overview}

組織層級鎖定目標可讓您在組織、環境和區域範圍中推出功能。 這可控制整個組織或特定部署環境（而非個別使用者）中的功能可用性。

此模式是針對企業級功能閘道和環境特定轉出所設計。

### 使用時機 {#org-level-when}

當您想要執行下列動作時使用組織層級鎖定目標：

* 為整個組織啟用功能
* 依環境控制功能可用性（例如，中繼與生產）
* 執行地區轉出
* 根據權益或訂閱層級閘道功能

### 限制 {#org-level-limitations}

組織和環境層級鎖定目標並非針對下列情況而設計：

* 不支援豐富型使用者設定檔型目標定位
* 不支援在個別使用者層級以百分比為基礎的轉出
* 不適用於實驗或A/B測試

## 比較 {#comparison}

| 維度 | 使用者層級鎖定目標 | 組織和環境層級鎖定目標 |
|---|---|---|
| 控制範圍 | 個別使用者 | 組織、環境、地區 |
| 目標定位基礎 | 使用者設定檔和屬性 | 組織和環境內容 |
| 轉出百分比 | 是 | 無 |
| A/B testing | 是 | 無 |
| 企業閘道 | 無 | 是 |

## 選擇正確的模式 {#choosing}

* 如果您的問題是&#x200B;*「哪些使用者應該看到此功能？」* →使用&#x200B;**使用者層級鎖定目標**
* 如果您的問題是&#x200B;*「哪些組織或環境應該有此功能？」* →使用&#x200B;**組織和環境層級鎖定目標**
