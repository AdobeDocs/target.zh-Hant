# 標幟簡介 {#introduction}

>[!AVAILABILITY]
>
>標幟目前位於Beta中，可供Adobe Target客戶使用。 請聯絡您的 Adobe 代表以請求存取權。

Adobe Target中的標幟是一種控管發行系統，可讓團隊將功能部署至生產環境，同時精確控制可檢視這些功能的人和時間。 功能可在生產環境中使用，對使用者不可見，然後針對目標對象逐步開啟，無需任何重新部署。

## 從開發到生產 {#dev-to-prod}

標幟支援功能從最早的開發階段到一般可用性的完整歷程：

1. **開發人員測試** — 開發人員建立功能標幟，將其程式碼部署到任何環境，並僅針對他們自己的工作階段進行測試。 不會影響其他使用者，也不需要分支。

2. **目標式預覽** — 產品擁有者會將對象連結至功能標幟，使該功能可供已定義的使用者子集（例如，測試版參與者或特定區域）使用以進行驗證和回饋。

3. **逐步推出** — 功能會逐步開放給更廣大的對象 — 1%、10%、50%、100% — 並可隨時暫停、調整或關閉功能。

4. **一般可用性** — 一旦驗證完成，所有使用者都可以使用該功能。

## 核心功能 {#capabilities}

旗標是一種功能管理平台，可提供：

* **功能旗標** — 在執行階段開啟或關閉目標對象的任何功能，而不需重新部署程式碼。

* **對象鎖定目標** — 控制哪些人能使用使用者設定檔資料、百分比式規則、電子郵件地址、電子郵件網域、IP位址或內容屬性看到功能。

* **功能群組** — 跨應用程式將多個相關功能標幟捆綁，並作為單一單元進行管理，以確保相同的對象看到一致的體驗。

* **版本** — 將來自多個團隊和應用程式的功能標幟分組為單一版本事件，以協調大型跨團隊轉出。

* **逐步轉出** — 階段功能遞增傳送，以降低風險、收集意見並管理後端負載。

* **刪除開關** — 在偵測到問題時立即關閉任何功能，不需要變更或重新部署程式碼。

* **AEP SDK支援** — 透過AEP Web SDK和AEP Mobile SDK部署旗標，以便在Web和行動應用程式間評估旗標。
