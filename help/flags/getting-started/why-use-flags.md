# 為何使用標幟 {#why-use}

每當您需要控制誰能看到功能以及何時看到時，旗標是正確的工具 — 無論您是在開發中進行測試、使用使用者子集進行驗證，還是跨多個團隊協調大型版本。

## 常見使用案例 {#use-cases}

**開發期間的選擇性測試**
針對您自己的工作階段測試功能，而其他開發人員則不受影響地繼續工作。 不需要複雜的程式碼分支。

**階段轉出並附使用者回饋**
先向一小群使用者發行功能。 收集意見反應、解決問題，然後在完整版發行前逐步擴大對象。

**逐步轉出至生產環境**
以漸進方式開啟新功能 — 先是1%、10%、50%，然後是100%的使用者。 監視每個步驟的效能和使用者回應。 如果發生錯誤，請立即將其關閉。

**後端載入管理**
逐步推出以避免後端服務突然出現流量尖峰，而非一次讓所有使用者看到新功能。

**協調的多重應用程式發行**
針對一組特定使用者，同時跨多個應用程式和團隊啟用一項功能。 標幟可確保整個發行表面的一致性。

**延期的發行**
提早將程式碼部署到生產環境，然後在精確的時間啟動功能 — 例如，在產品啟動事件開始時 — 沒有在最後一刻變更任何程式碼。

**終止引數**
如果發行後發現問題，請立即關閉該功能，而不使用Hotfix或重新部署。
