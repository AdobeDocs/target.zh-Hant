# 限制{#limits}

有關 Adobe Target 中會影響活動和其他元素之字元限制與其他限制(選件大小、對象、設定檔、值、參數等) 的資訊。

下列限制是建議的限制。當接近或超過這些限制時，效能可能變慢。介面載入時間緩慢，也可能是由複雜的活動造成，例如一個活動中有許多對象、目標和體驗。

複雜度高的活動應由 Adobe 諮詢服務審閱過，並先在有限制的環境中測試過，再正式上線。

## 活動名稱

**限制**：250個字元。

## 對象名稱

**限制**：256個字元。

超過 256 個字元的值會遭截斷。

## categoryId 參數

**限制**：250個字元。

## 客戶屬性名稱

**限制**：128個字元。

## 客戶屬性別名ID

**限制** 50個字元。

## 實體自訂屬性

**限制**：最大長度視語言而定。

* 15,000個字元(單一值、一個和雙位元組語言)
* 500個值，每個值100個字元(多值)

單一值實體自訂屬性的長度上限為15,000個字元(對於一個和雙位元組UTF-8編碼語言，例如英文和其他拉丁指令碼)或10,000個字元(對於三位元組UTF-8編碼語言，例如中文、日文和韓文)。

多值實體自訂屬性可包含超過500個值。每個個別值限定為100個字元。所有值的字元總數必須符合單一值實體自訂屬性長度上限的限制(請參閱上文)。

## entityId參數

**限制**：1,000個字元。

## ExcludedIDs {#excludedid}

**限制**：POST要求需要5KB。2,083字元減去GET請求的URL長度。

對於GET請求，雖然後端的限制為5KB，但由於Microsoft Internet Explorer將URL限制為2,083個字元，因此實際限制是2,083個字元減去URL的目前長度。

## 體驗名稱

**限制**：20個字元。

## In-mbox描述檔屬性值

**限制**：256個字元。

超過限制的值會遭截斷。

## mbox請求中的In-mbox描述檔

**限制**：50份個人檔案。

第 50 個之後的描述檔全部會被忽略。

## In-mbox描述檔名稱

**限制**：128個字元。

## mbox名稱

**限制**：250個字元。

## mbox 參數

**限制**：下列限制適用於mbox參數：

* mbox參數：每個mbox500個參數。
* 描述檔參數：500個參數。
* 每個 mbox 的設定檔參數:
* 其他參數 (URL、轉介 URL 等): 各參數類型每個 mbox 50 個。

對於要登入Target資料庫的參數，上述限制適用於標準mbox請求。除非要求因網頁瀏覽器限制而縮短，否則適用上述限制。

如果您在 Mobile Services SDK 使用[批次傳送 API](https://developers.adobetarget.com/api/#server-side-batch-delivery)，50 個 mbox 參數、50 個描述檔參數和 50 個其他參數類型限制，即為 API 本身限制。無法使用批次傳送 API，傳送超過上述數量限制的要求。如果要求超過上述限制，API 會傳回下列錯誤訊息:「mboxParameters 數量不得超過 100。」

## mbox請求URL

**限制**：2,083個字元。

此限制是由於Microsoft Internet Explorer URL長度限制所致。

## mbox3rdPartyId 參數

**限制**：60個字元。

## 選件名稱

**限制**：250個字元。

## 選件大小

**限制**：下列大小限制適用於選件：

* HTML 選件為 256 KB.
* 來自 UI 的視覺選件為 64 KB.
* 來自 API 512 KB.

如果您使用全域mbox，則限制頁面傳回的整組內容。限制選件大小可改進頁面載入時間。如果超過限制，會顯示下列訊息: 

「體驗的內容過大，無法傳送。請修改體驗，降低影響的頁面代碼。」

## orderId 參數

**限制**：120個字元。

建議的限制。

## orderTotal 參數

**限制**：120個字元。

建議的限制。

## productPurchasedId 參數

**限制**：每個逗號分隔值47個字元。

系統會被截斷。

## 可重複使用的對象/帳戶

**限制**：75個受眾。

建議的限制。如果太多，介面會發生 JavaScript 逾時。

## Target UI中的指令碼描述檔輸入方塊

**限制**：2,000個字元。

建議的限制。依編碼字串的大小而定，可能比原始字串大許多。如果字串太長，還未送到 Adobe Target 就會失敗。

## 指令碼描述檔名稱

**限制**：50個字元。

## 指令碼描述檔值

**限制**：2,048個字元。

基於效能考量，我們建議傳回的值不要超過 256 個字元。

對於字串傳回值，如果傳回值大小超過2,048個字元，系統會停用指令碼。

對於陣列傳回值，如果陣列的串連值大小超過2,048個字元，系統會停用指令碼。

## 目標條件

**限制**：1,000個值。

建議的限制。這是指定位文字區域中以行分隔之值的數目。例如，在郵遞區號定位中輸入 1,000 個郵遞區號。
