## 它有什麼作用？  {#section_4ECAACC68723402EB3649033190E1BBC}

| 活動類型 | 詳細資料 |
|--- |--- |
| 手動 [!UICONTROL A/B測試]<P>![圖示](/help/main/c-activities/assets/icon_ab.png) | 比較兩個或多個體驗，以檢視在預先指定的測試期間，哪個體驗最能改善轉換。<P>如需詳細資訊，請參閱 [A/B 測試](/help/main/c-activities/t-test-ab/test-ab.md)。 |
| [!UICONTROL 自動分配]<P>![圖示自動分配](/help/main/c-activities/assets/icon_auto_allocate.png) | 從兩個或多個體驗中識別獲勝者，然後將流量重新導向獲勝者體驗，隨著測試執行和學習增加轉換。<P>如需詳細資訊，請參閱[自動分配](/help/main/c-activities/automated-traffic-allocation/automated-traffic-allocation.md)。 |
| [!UICONTROL 自動鎖定目標]<P>![圖示在](/help/main/c-activities/assets/icon_auto_target.png) | 使用進階機器學習來個人化內容，並透過找到多個高執行效能、行銷人員定義的體驗，然後根據訪客的個別客戶設定檔與類似訪客先前的行為，提供訪客量身打造的最佳體驗，藉此促進轉換。<P>如需詳細資訊，請參閱「[自動鎖定目標以獲得個人化體驗](/help/main/c-activities/auto-target/auto-target-to-optimize.md)」。 |
| [!UICONTROL Automated Personalization] (AP)<P>![圖示AP](/help/main/c-activities/assets/icon_ap.png) | 使用進階機器學習來個人化內容，並透過結合特定選件或訊息來促進轉換，然後根據訪客的個別客戶設定檔比對訪客的不同選件變數。<P>如需詳細資訊，請參閱 [Automated Personalization](/help/main/c-activities/t-automated-personalization/automated-personalization.md)。 |
| [!UICONTROL 多變數測試] (MVT)<P>![MVT圖示](/help/main/c-activities/assets/icon_mvt.png) | 比較頁面上元素間選件的組合，以檢視哪個選件組合對特定對象的執行效果最佳。 此外，找出頁面的哪個元素在整個預先指定的測試期間最能改善轉換。<P>如需詳細資訊，請參閱[多變數測試](/help/main/c-activities/c-multivariate-testing/multivariate-testing.md)。 |
| 體驗鎖定 (XT)<P>![圖示XT](/help/main/c-activities/assets/icon_xt.png) | 根據一組市場行銷人員定義的規則和條件為特定對象傳送內容。<P>如需詳細資訊，請參閱[體驗鎖定目標](/help/main/c-activities/t-experience-target/experience-target.md)。 |

## 您為何使用此型別的活動？ {#section_46A70DD7CE3448749E635DDF5EAFC131}

| 活動類型 | 原因 |
|--- |--- |
| 手動 [!UICONTROL A/B測試] | 高度控制的實驗會具有流量測量，依百分比而非規則分割，可讓您分析測試資料，收集關於您的對象的分析資料，並會決定哪個體驗的執行最佳。 |
| [!UICONTROL 自動分配] | 識別成功體驗並調整流量分配的方式，以儘快將成功體驗傳送給訪客，支援更快且更高的轉換可能性。 |
| [!UICONTROL 自動鎖定目標] | 在多個體驗中識別獲勝者，然後傳送最適當的體驗給特定訪客的方式。鎖定目標會隨著時間適應訪客興趣的變更，因為演算法會預測在特定時間某些體驗訪客的轉換傾向。 |
| [!UICONTROL Automated Personalization] (AP) | 一種方式是將一組選件 (已在單頁或多個頁面上的元素中建立或預先定義) 個人化，並傳送最佳選件組合以吸引特定訪客。 |
| [!UICONTROL Multivariate Testing] (MVT) | 一種在多個元素中顯示多個選件，然後針對特定目標同時測試所產生的不重複體驗的方式，這有助於判斷哪個元素變數最成功。 MVT活動也可顯示哪些元素對訪客的互動具有最大的正面或負面影響。 |
| [!UICONTROL 體驗鎖定] (XT) | 根據一組已定義的配置規則將特定內容鎖定在特定對象的方法。 |

## 哪型別行銷人員應該使用活動型別？ {#section_A843D663D3E543FFB1A594266B560395}

| 活動類型 | 行銷人員 |
|--- |--- |
| 手動 [!UICONTROL A/B測試] | 在統計資料方面有知識。<P>有時間等候測試期間結束再分析結果。 |
| [!UICONTROL 自動分配] | 時間很有限。<P>必須識別最佳體驗並快速傳送。<P>想要可以在測試執行時「窺視」結果。 |
| [!UICONTROL 自動鎖定目標] | 有數個符合資格的體驗。<P>想要根據訪客的動態和變動的設定檔，以最佳時間配對體驗與特定訪客。 |
| [!UICONTROL Automated Personalization] (AP) | 有一或多個選件。<P>想要針對眾多具有唯一設定檔和行為的特定訪客，建立可產生最佳個人化體驗的選件組合。 |
| [!UICONTROL Multivariate Testing] (MVT) | 在統計資料方面有知識。<P>有一或多個選件。<P>想要分析與頁面元素互動相關的轉換趨勢。 |
| [!UICONTROL 體驗鎖定] (XT) | 必須傳遞特定體驗或內容片段給特定對象。 |

## 統計詳細資料 {#section_22CF2D07DB054505AB5EC702B99A5BB0}

| 活動類型 | 詳細資料 |
|--- |--- |
| 手動 [!UICONTROL A/B測試] | 此測試會將每個挑戰者體驗與控制體驗進行比較，然後對所有體驗的效能進行排名，識別與控制體驗相較之下的成功體驗和失敗體驗。 |
| [!UICONTROL 自動分配] | 測試會立即對真正的獲勝者產生統計保證，然後將更多流量導向對該成功體驗而言有較高轉換可能性的對象。 |
| [!UICONTROL 自動鎖定目標] | 最佳化機制會隨時間推移顯示提升度的增加和減少，並在決定要將哪個體驗提供給哪個訪客之前，識別每個體驗的相關對象。 最佳化機制會透過轉換、區段、引數和設定檔指令碼來告知。 從那裡，機制會自動選擇使用哪個演演算法來產生較高的提升度和轉換率。 |
| [!UICONTROL Automated Personalization] (AP) | 最佳化機制會根據新訪客行為和類似訪客過去的行為，對並行的控制群組測量選件的效能，不斷地調整要將哪些體驗傳送至哪些訪客。 |
| [!UICONTROL Multivariate Testing] (MVT) | 測試有助於發掘特定元素對轉換的相關影響。 |
| [!UICONTROL 體驗鎖定] (XT) | 此方法定義的規則會將特定體驗或特定內容片段鎖定在特定對象。客戶可以在體驗層級進行更新。 |

## 優點與考量事項 {#section_56C46ABEF7B945DDA0C1E6D714377123}

| 活動類型 | 福利 | 考量事項 |
|--- |--- |--- |
| 手動 [!UICONTROL A/B測試] | A/B休息可協助您全面瞭解每個體驗的表現，不僅僅是哪些體驗表現最佳。 | 在 [!UICONTROL A/B測試]，如果您在樣本量達到之前檢視測試結果，您可能會依賴不準確的結果（您無法「偷看」更早！）。<P>取消讚 [!UICONTROL 自動分配]，在A/B測試中，即便在您發現有些體驗勝過其他體驗之後，流量分配仍會維持固定。<P>如需以下專案的最佳實務相關資訊： [!UICONTROL A/B測試] 活動，請參閱 [A/B測試該執行多久？](/help/main/c-activities/t-test-ab/sample-size-determination.md) 和 [十個常見的A/B測試陷阱和避免方法](/help/main/c-activities/t-test-ab/common-ab-testing-pitfalls.md). |
| [!UICONTROL 自動分配] | [!UICONTROL 自動分配可減少一般 A/B 測試的成本，因為它相較於手動 A/B 測試具有較高的整體轉換率。]轉換率比較高，因為 [!UICONTROL 自動分配] 會將更多流量推送至最高效能的體驗，這表示您可以在測試期間結束前就意識到成功體驗的好處（您可以偷看！）。 | [!UICONTROL 自動分配可識別獲勝者，但無法在失敗者之間加以區分。]如果您必須知道每個體驗的執行方式，建議您使用A/B測試。<P>此 [!UICONTROL 自動分配] 此功能只對一個進階量度設定產生作用，即「增加計數以及讓使用者留在活動中」。 如果您不想計算重複的轉換次數，應改用A/B測試。 |
| [!UICONTROL 自動鎖定目標] | 替換為 [!UICONTROL 自動鎖定目標]，機器學習會套用至任何型別的體驗，包括多頁體驗。 一個 [!UICONTROL 自動鎖定目標] 活動也可讓您取得 [!UICONTROL Automated Personalization] 同時使用熟悉的A/B測試工作流程。 | 替換為 [!UICONTROL 自動鎖定目標]，如果您想要經常或經常變更優惠方案的內容，演演算法需要在每次變更後有足夠的時間加以利用，才能學到內容，並將內容傳送給適當的訪客。 |
| [!UICONTROL Automated Personalization] (AP) | 替換為 [!UICONTROL Automated Personalization]，您可以在一個位置收集所有優惠方案，而演演算法會決定優惠方案的最佳組合。 您不需指定或建置個別的體驗。[!UICONTROL Automated Personalization] 使用與相同的機器學習演演算法 [!UICONTROL 自動鎖定目標]. | 當您結合多個選件時會發生組合上的爆增，因而導致需要龐大的流量。此 [!UICONTROL Automated Personalization] 演演算法會考慮許多因素；因此，需要最多的流量。<P>[!UICONTROL Automated Personalization] 無法在中使用報表 [目標分析](/help/main/c-integrating-target-with-mac/a4t/a4t.md) (A4T)。 |
| [!UICONTROL Multivariate Testing] (MVT) | 替換為 [!UICONTROL Multivariate Testing]，您可以同時測試多個元素。 | A [!UICONTROL 多變數測試] 相當耗時，而且由於有許多變數在運作，不一定能令人放心地產生成功體驗。<P>要達到完成測試所需的流量經常充滿挑戰。因為所有 [!UICONTROL 多變數測試] 實驗是全階乘，一次太多變更的元素可能會快速增加許多必須測試的可能組合。<P>即便是具有非常高流量的網站，也可能難以在適當的時間內完成具有超過 25 個組合的測試。 |
| [!UICONTROL 體驗鎖定] (XT) | 替換為 [!UICONTROL 體驗鎖定]，您可以根據任何活動結果推斷出的見解快速採取行動。<P>例如，如果您執行A/B測試時，其中的挑戰者並未優於控制，但結果指出特定訪客區段對挑戰者的轉換比控制多四倍，則您可使用 [!UICONTROL 體驗鎖定] 將挑戰者體驗導向該特定區段。 | [!UICONTROL 體驗鎖定目標不允許您控制跨多個對象間體驗的分割百分比。] |
