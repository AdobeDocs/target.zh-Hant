# 成功量度

在 [!DNL Adobe Target] 成功度量是用於衡量活動成功與否的參數。 成功度量包括關鍵業務量度，使您能夠確定給定體驗或在 [!DNL Target] 的子菜單。

例如，您可以判斷新選件是否會提高每位訪客帶來的收入，或將項目新增至購物車的機會。成功量度非常適合用於探索關於註冊、訂購或購買漏斗的問題，但也適合探索關於訪客或客戶參與度的問題。

## 總覽

在 [!DNL Target]，成功度量是預配置的，它具有用於報告和跟蹤目的的最佳選項。

預設情況下，轉換事件設定為「」[!UICONTROL 增量計數並保留活動中的用戶]&quot; 轉換只計算一次，不計算重複轉換，訪問者始終看到活動內容。

設定為「」的收入度量[!UICONTROL 增量計數並保留活動中的用戶]&quot;僅記錄同一訪問者所做的第一次訂單的詳細資訊。 所有後續訂單都會增加轉換計數，但不會將收入添加到RPV/AOV/Sales中，並且不會包括在 [!UICONTROL 訂單詳細資訊] 報告。

>[!NOTE]
>
>對於使用 [分析作為報告源](/help/main/c-integrating-target-with-mac/a4t/a4t.md) (A4T)，目標度量將始終使用「[!UICONTROL 增量計數和保留活動中的用戶]&quot;和&quot;[!UICONTROL 論每個印象]。 這是 *不* 可配置。

下列為可使用的成功量度:

| 成功量度 | 測量方式 | 定義 |
|--- |--- |--- |
| 轉換 | 轉換型 | 轉換是指訪問者對您定義的站點執行操作，如 <ul><li>已按一下按鈕</li><li>已檢視頁面</li><li>完成調查</li><li>已購買</li></ul>每個訪問者或每次訪問者完成轉換時，都可以計算一次轉換。 |
| 收入 | 轉換型 | 造訪產生的收入。您可以從以下收入量度中選擇: <ul><li>每位訪客帶來的收入 (RPV)</li><li>平均訂購值 (AOV)</li><li>銷售總額</li><li>訂單</li></ul> |
| 頁面檢視 | 參與型 | 每個不重複訪客都會被計為轉換。 |
| 自訂得分 | 參與型 | 從訪問者第一次看到活動的首次顯示時起，基於分配給站點上訪問的頁面的值的聚合分數 [!DNL Target] 請求。 |
| 網站逗留時間 | 參與型 | 從訪問者看到活動的首次顯示開始，在訪問中花費的時間（以秒為單位） [!DNL Target] 請求載入最終頁面，並在會話中發出請求。 |

若為參與型量度 (非以轉換和收入為基礎的量度)，訪客每次瀏覽均須重新取得活動資格，以增加該工作階段計數。相關量度會在重新取得資格後開始增加計數，並在各訪客的工作階段結束後停止。閒置 30 分鐘後，工作階段就會結束。因此，在測試過程中，您不會立即看到結果；但是，該會話的所有結果在會話結束後的幾分鐘內都可用。

## 自定義成功度量

您也可以建立自訂成功量度。

選取成功量度之後，請選取訪客為了達成目標採取的動作。例如，選擇 [!UICONTROL 轉換] 度量，將其設定為每個訪問者一次計數，然後設定訪問者查看特定頁面（或一組頁面）時是否成功 [!DNL Target] 請求，或按一下特定連結。

如果啟用， [!UICONTROL 一次轉換的估計值] 欄位(不適用於 [!UICONTROL 頁分] 度量)為您的目標提供值，但不為其他度量提供值。 此值可讓 [!DNL Target] 計算收入中預估的提升度。此欄位為可選; 不過，任何非收入量度的遞增收入若沒有它即無法計算。對於所有收入指標([!UICONTROL 每訪客收入]。 [!UICONTROL 平均訂單值]。 [!UICONTROL 銷售總額], [!UICONTROL 訂單])，估計使用 [!UICONTROL 每訪客收入]。 資料類型為貨幣。請參閱[預估收入中的提升度](/help/main/administrating-target/r-target-account-preferences/estimating-lift-in-revenue.md)以取得詳細資訊。

為您的活動選擇的成功量度可在您檢視活動的報表時，於報表設定中取得。

某些指標，如 [!UICONTROL 自定義計分] 和 [!UICONTROL 每訪客收入]，需要傳遞訂單合計和訂單ID等資訊的自定義實現。

## 進階設定 {#section_7CE95A2FA8F5438E936C365A6D43BC5B}

使用進階設定來管理測量成功的方式。選項包括添加依賴項、選擇是將用戶保留在活動中還是刪除它們，以及是對每個參與者或每個印象計數一次度量。

訪問 [!UICONTROL 高級設定] 選項，按一下 **[!UICONTROL 垂直橢圓]** > **[!UICONTROL 高級設定]**。

![進階設定功能表](/help/main/c-activities/r-success-metrics/assets/advanced-settings.png)

>[!NOTE]
>
>如果您使用 [!DNL Adobe Analytics] 作為您的報表來源，則設定是由 [!DNL Analytics] 伺服器管理。的 [!UICONTROL 高級設定] 選項。 有關詳細資訊，請參見 [Adobe Analytics為Adobe Target(A4T)的報告來源](/help/main/c-integrating-target-with-mac/a4t/a4t.md)。

### 添加依賴項

您可以使用高級設定建立相關成功度量，僅當訪問者首先到達另一個度量時才遞增一個度量。

![新增相依性](/help/main/c-activities/r-success-metrics/assets/UI_dep_success_metric.png)

例如，測試轉換可能只有在轉換之前訪客點擊了選件，或是達到特定頁面時才有效。

依賴關係功能是 *不* 支援以下功能：

* [!UICONTROL Recommendations] 活動. 所有其他活動類型均支援此功能。
* 如果您使用 [分析作為報告源](/help/main/c-integrating-target-with-mac/a4t/a4t.md) (A4T)。
* 「已檢視頁面」度量類型。
* 可視化體驗撰寫器 (VEC) 活動的「按一下元素」度量類型。

相依成功量度在下列情況下不會轉換:

* 如果您建立循環相依性，其中的 metric1 相依於 metric2，而 metric2 相依於 metric1，則這兩個量度都不會轉換。
* 自動個人化活動會在達到轉換量度時釋出使用者並重新開始活動，因此相依於轉換量度的任何量度永遠不會轉換。

### 使用者達到此目標量度後會發生什麼事?

使用進階設定來判斷使用者達到目標量度之後要執行的動作。下表顯示了可用選項：

| 使用者達到此目標量度之後 | 選項 |
|--- |--- |
| [!UICONTROL 增加計數以及讓使用者留在活動中] | 指定計數的遞增方式:<ul><li>每次進入則計數加一 (預設)</li><li>在每次曝光時，排除頁面重新整理</li><li>在每次曝光時</li></ul> |
| [!UICONTROL 增量計數、釋放用戶和允許再入] | 選取訪客重新進入活動時會看見的體驗:<ul><li>相同體驗 (預設)</li><li>隨機體驗</li><li>未見的體驗</li></ul> |
| [!UICONTROL 再入增量計數、釋放用戶和欄] | 決定使用者看見的內容而非活動內容:<ul><li>相同體驗，無追蹤 (預設)</li><li>預設內容，或其他活動內容</li></ul> |

>[!NOTE]
>
>如果將度量配置為 [!UICONTROL 增量計數] 選項（上面提到），度量計數只在訪問者級別每個進入者正確遞增一次。 在訪問級別，每次訪問一次新會話的度量計數增量。

### 計數將如何增加:

選擇所需的行為：

* 每個參與者一次
* 在每個印象中（排除頁面刷新）
* 在每次曝光時
