# ![PREMIUM](/help/assets/premium.png) 使用動態和靜態包含規則

有關在[!DNL Adobe Target]中建立標準和促銷的包含規則，以及新增動態或靜態篩選規則，以便為建議取得更佳結果的資訊。

對於條件和促銷活動，建立和使用包含規則的過程很相似，就如同使用案例和範例很相似一樣。本節將涵蓋標準和促銷活動以及包含規則的使用。

## 將篩選規則新增至條件 {#section_CD0D74B8D3BE4A75A78C36CF24A8C57F}

[建立條件時](/help/c-recommendations/c-algorithms/create-new-algorithm.md#task_8A9CB465F28D44899F69F38AD27352FE)，請按一下&#x200B;**[!UICONTROL 「包含規則」]**&#x200B;下的&#x200B;**[!UICONTROL 「新增篩選規則」]**。

![](assets/inclusion_options_new.png)

可用的選項依選取的行業別和建議金鑰而不同。

## 將篩選規則新增至促銷活動  {#section_D59AFB62E2EE423086281CF5D18B1076}

[建立促銷活動](/help/c-recommendations/t-create-recs-activity/adding-promotions.md#task_CC5BD28C364742218C1ACAF0D45E0E14)時，請選取&#x200B;**[!UICONTROL 「依屬性促銷」]**，然後按一下&#x200B;**[!UICONTROL 「新增篩選規則」]**。

![](assets/inclusion_options.png)

## 篩選類型 {#section_0125F1ED10A84C0EB45325122460EBCD}

以下各節列出[!UICONTROL 動態篩選]和[!UICONTROL 依值篩選]的篩選選項類型，供標準和促銷使用：

### 動態篩選

動態包含規則比靜態包含規則更強大，而且可產生更佳的結果和參與度。 考慮以下事項:

* 動態包含規則會比對使用者描述檔參數或mbox呼叫中的屬性，以提供建議。

   例如，您可以建立「最受歡迎的准則」建議。 從傳回的建議集合中，您可以根據使用者存取顯示建議之頁面時傳遞的屬性，即時篩選任何建議。

* 使用靜態規則來限制建議中包含的項目（而非使用系列）。

* 您可以視需要建立任意數量的動態包含規則。 包含規則是以 AND 運算子來結合。必須符合所有規則，才能在建議中納入某個項目。

動態篩選可使用下列選項：

| 動態篩選選項 | 詳細資料 |
| --- | --- |
| [實體屬性比對](/help/c-recommendations/c-algorithms/entity-attribute-matching.md) | 透過比較潛在建議項目群組與使用者已互動的特定項目，動態篩選。<br>當您 [!UICONTROL 想要顯] 示最有可能吸引訪客的建議（例如訪客的最愛品牌）時，請使用實體屬性符合。 |
| [設定檔屬性比對](/help/c-recommendations/c-algorithms/profile-attribute-matching.md) | 透過比較項目（實體）與使用者描述檔中的值，動態篩選。<br>當您 [!UICONTROL 想要顯示] 符合訪客描述檔中儲存之值（例如大小或我的最愛品牌）的建議時，請使用描述檔屬性符合。 |
| [參數比對](/help/c-recommendations/c-algorithms/parameter-matching.md) | 透過比較項目（實體）與請求中的值（API或mbox），動態篩選。<br>使用 [!UICONTROL 參數] 匹配可建議符合頁面參數或訪客參數的內容，例如裝置尺寸或地理位置。 |

### 按值篩選

以下選項可用於按值篩選：

| 依值篩選選項 | 詳細資料 |
| --- | --- |
| [靜態濾鏡](/help/c-recommendations/c-algorithms/static-value.md) | 手動輸入一或多個靜態值以進行篩選。 |

## 可用運算子{#operators}

動態標準和促銷比靜態標準和促銷強大得多，可產生更佳的結果和參與度。

以下範例提供如何在行銷活動中使用動態促銷和排除的一般概念：

| 運算元 | 範例 |
| --- | --- |
| Equals<br>（適用於實體屬性比對、描述檔屬性比對、參數比對和靜態篩選）。 | 在動態促銷中使用「等於」運算子，當訪客在您的網站上檢視項目（例如產品、文章或影片）時，您可從以下網站促銷其他項目：<ul><li>同一品牌</li><li>相同類別</li><li>同一類別AND來自房屋品牌</li><li>同一家商店</li></ul> |
| 不等於<br>（適用於實體屬性比對、描述檔屬性比對、參數比對和靜態篩選）。 | 在動態促銷中使用「不等於」運算子，當訪客在您的網站上檢視項目（例如產品、文章或影片）時，您可以從以下位置促銷其他項目：<ul><li>不同的電視劇</li><li>另一種類型</li><li>不同的產品系列</li><li>不同的樣式ID</li></ul> |
| 不包含子字串<br>（可用於實體屬性比對、描述檔屬性比對、參數比對和靜態篩選）。 | 使用「不包含子字串」運算子，當訪客在您的網站（例如產品）上檢視項目時，您可以促銷其他符合下列條件的項目：<ul><li>標題不包含粗體字</li></ul> |
| 開頭為<br>（可用於實體屬性比對、描述檔屬性比對、參數比對和靜態篩選）。 | 使用「開頭為」運算子，當訪客在您的網站上檢視項目（例如產品）時，您可以促銷其他符合下列條件的項目：<ul><li>產品名稱以iPhone開頭</li></ul> |
| 結尾為<br>（適用於實體屬性比對、描述檔屬性比對、參數比對和靜態篩選）。 | 使用「結尾於」運算子，當訪客在您的網站上檢視項目（例如產品）時，您可以促銷其他符合下列條件的項目：<ul><li>內容以EN結尾，表示英文</li></ul> |
| 大於或等於<br>（適用於實體屬性比對、描述檔屬性比對、參數比對和靜態篩選）。 | 使用「大於或等於」運算子，當訪客在您的網站（例如產品）上檢視項目時，您可以促銷其他符合下列條件的項目：<ul><li>成本相同或更貴</li></ul> |
| 小於或等於<br>（適用於實體屬性比對、描述檔屬性比對、參數比對和靜態篩選）。 | 使用「小於或等於」運算子，當訪客在您的網站（例如產品）上檢視項目時，您可以促銷其他符合下列條件的項目：<ul><li>成本相同或較便宜</li><li>排除較便宜的項目</li></ul> |
| 介於<br>（可用於實體屬性比對、描述檔屬性比對和參數比對）。 | 在動態促銷中使用「介於」運算子，當訪客在您的網站上檢視項目（例如產品、文章或影片）時，您可以促銷其他項目：<ul><li>更貴</li><li>更便宜</li><li>成本正負30%</li><li>同一季的後續劇集</li><li>系列中的先前書籍</li></ul> |
| 包含在List<br>中（可用於描述檔屬性比對和參數比對）。 | 使用描述檔屬性比對中的「包含在清單中」運算子，當訪客在您的網站上檢視項目（例如產品、文章或影片）時，您可以促銷其他項目：<ul><li>在訪客的地理位置中可用</li></ul>**範例**:您只想建議訪客的地理區域中可用的項目。<br>您的篩選規則可能如下所示：<br>`availableGeographies list contains an item in user.currentGeography`<br>**附註**:使用此運算子時，規則的右側 [會](#caveats) 出現清單。 |
| 不包含在清單<br>中（可用於描述檔屬性比對和參數比對）。 | 使用描述檔屬性比對中的「不包含在清單中」運算子，當訪客在您的網站上檢視項目（例如產品、文章或影片）時，您可以排除其他項目：<ul><li>在訪客已檢視的前十個項目清單中</li></ul></ul>**範例**:您不想促銷訪客最近檢視且不感興趣的項目。<br>您的篩選規則可能如下所示：<br>`id is not contained in list user.lastViewedItems`<br>**附註**:使用此運算子時，規則的右側 [會](#caveats) 出現清單。 |
| 清單包含<br>中的項目（可搭配實體屬性比對、描述檔屬性比對和參數比對使用）。 | 使用描述檔屬性比對中的「清單包含項目」運算子，當訪客在您的網站上檢視項目（例如運動活動或音樂會）時，您可以促銷其他項目：<ul><li>與其中一個訪客的最愛團隊相關聯</li></ul>**範例**:您想要建議與訪客其中一個最愛團隊關聯的遊戲。<br>您的篩選規則可能如下所示：<br>` teamsPlaying list contains an item in user.favoriteTeams`<br>**附註**:使用此運算子時，規則的兩側都 [會](#caveats) 出現清單。 |
| 清單中不包含項目<br>（可搭配實體屬性比對、描述檔屬性比對和參數比對使用）。 | 在參數屬性比對中使用「清單不包含項目」運算子，當訪客在您的網站上檢視項目（例如產品、文章或影片）時，您可以排除其他項目：<ul><li>列在禁止類型清單中</li></ul>**範例**:您想要排除成人訪客可使用的項目，例如煙草和酒精。<br>您的篩選規則可能如下所示：<br>`itemType is not contained in list mbox.prohibitedTypes`<br>**附註**:使用此運算子時，規則的兩側都 [會](#caveats) 出現清單。 |
| 清單包含<br>中的所有項目（可搭配實體屬性比對、描述檔屬性比對和參數比對使用）。 | 使用描述檔屬性比對中的「清單包含所有項目」運算子，當訪客在您的網站上檢視項目（例如工作貼文或方式）時，您可以促銷其他符合下列條件的項目：<ul><li>包含一組技能</li><li>包含一組必要的配料</li></ul>**範例1**:假設訪客擁有一組技能（Java、C++和HTML）。目錄中的項目是具備必要技能的工作（Java和HTML）。 您要先確定訪客的描述檔包含所有必要技能，再將工作推薦給訪客。<br>您的篩選規則可能類似：<br>`profile.jobSeekerSkills contains all items in entity.requiredSkills`<br>**範例2**:假設使用者有食品配料清單。食譜包含所需的配料清單。 您要先確定訪客的描述檔包含所有必要的成分，再將配方建議給訪客。<br>您的篩選規則可能如下所示：<br>`profile.ingredientsInPantry contains all items in recipe.ingredientsRequired`<br>**附註**:使用此運算子時，規則的兩側都 [會](#caveats) 出現清單。 |
| 清單中不包含<br>中的所有項目（適用於實體屬性比對、描述檔屬性比對和參數比對）。 | 使用實體屬性比對中的「清單不包含所有項目」運算子，當訪客在您的網站上檢視項目（例如體育賽事或音樂會）時，您可以促銷其他符合下列條件的項目：<ul><li>不包含一組團隊</li></ul>**範例**:假設體育賽事包括兩支球隊：休斯敦太空人隊和密爾沃基釀酒人隊。訪客的描述檔指出此訪客不想檢視這些團隊的遊戲。 如果這些團隊在比賽，您想確保不建議進行比賽。<br>您的篩選規則可能如下所示：<br>`profile.leastfavoriteTeams does not contain all items in entity.teamsPlaying`<br>**附註**:使用此運算子時，規則的兩側都 [會](#caveats) 出現清單。 |

## 按實體屬性匹配、描述檔屬性匹配和參數匹配{#section_7D30E04116DB47BEA6FF840A3424A4C8}篩選時處理空值

在依[!UICONTROL 「實體屬性符合]」、[!UICONTROL 「描述檔屬性符合]」和[!UICONTROL 退出標準和促銷活動的「參數符合]」篩選時，您可以選擇數個選項來處理空值。

以前，如果值空白，則不會傳回結果。如果條件有空白值，則「如果 *x* 為空白」下拉式清單可讓您選擇適當的動作來執行，如下圖所示:

![](assets/empty_value.png)

若要選取所需的動作，請將游標停留在齒輪圖示上 (![](assets/icon_gear.png))，然後選擇所需的動作:

| Action | 可用於 | 詳細資料 |
|--- |--- |--- |
| [!UICONTROL 忽略此篩選規則] | [!UICONTROL 描述檔屬性] 比對與 [!UICONTROL 參數比對] | 此動作是[!UICONTROL 描述檔屬性符合]和[!UICONTROL 參數符合]的預設動作。<br>此選項會指定忽略規則。例如，假設有三個篩選規則，而第三個規則未傳遞任何值，為了避免不傳回任何結果，您可以直接用空白值忽略第三個規則。 |
| [!UICONTROL 不要顯示此准則的任何結果]<br>（僅限准則） | [!UICONTROL 實體屬性匹配]、描述 [!UICONTROL 檔屬性匹配]、參數 [!UICONTROL 匹配] | 此動作是[!UICONTROL 實體屬性符合]的預設值。<br>此動作是新增 [!DNL Target] 此選項前如何處理空值的方式：此准則不會顯示任何結果。 |
| [!UICONTROL 不促銷任何項目<br>（僅限促銷）] | [!UICONTROL 實體屬性匹配]、描述 [!UICONTROL 檔屬性匹配]、參數 [!UICONTROL 匹配] | 此動作是[!UICONTROL 實體屬性符合]的預設值。<br>此動作是新增 [!DNL Target] 此選項前如何處理空值的方式：此准則不會顯示任何結果。 |
| [!UICONTROL 使用靜態值] | [!UICONTROL 實體屬性匹配]、描述 [!UICONTROL 檔屬性匹配]、參數 [!UICONTROL 匹配] | 如果值為空白，您可以選擇使用靜態值。 |

## 注意事項 {#caveats}

>[!IMPORTANT]
>
>使用「等於」和「不等於」運算子時，在執行階段，動態條件或促銷活動中的不同資料類型可能不相容。如果左側具有預先定義的屬性或自訂屬性，請在右側明智地使用[!UICONTROL Value]、[!UICONTROL Margin]、[!UICONTROL Inventory]和[!UICONTROL Environment]值。

![](assets/left_right.png)

下表顯示有效規則，以及在執行階段可能不相容的規則:

| 相容規則 | 可能不相容的規則 |
|--- |--- |
| value - 介於 - 目前項目的 90% 和 110% - salesValue | salesValue - 介於 - 目前項目的 90% 和 110% - value |
| value - 介於 - 目前項目的 90% 和 110% - value | clearancePrice - 介於 - 目前項目的 90% 和 110% - margin |
| margin - 介於 - 目前項目的 90% 和 110% - margin | storeInventory - 等於 - 目前項目的 - inventory |
| inventory - 等於 - 目前項目的 - inventory |  |
